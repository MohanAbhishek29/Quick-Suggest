About The Project

A High-Performance Autocomplete Engine demonstrating Advanced Data Structures.

🚀 Project Overview

Quick-Suggest is not just a simple dictionary; it is a demonstration of algorithm optimization and low-level system programming. Unlike standard web searches that rely on indexed databases (like SQL), this project implements a custom Trie (Prefix Tree) data structure in C++ to achieve near-instantaneous search results.

🧱 The Trie Data Structure

At the core of this engine lies the Trie. A Trie is a tree-like data structure used for storing strings. Each node represents a character, and paths from the root to leaf nodes represent words. 

struct TrieNode {
    unordered_map<char, TrieNode*> children;
    bool isEndOfWord;
    string description;
};

Why Trie? The time complexity for searching a word is O(L), where L is the length of the word. This is significantly faster than O(N) linear search or O(log N) binary search on huge datasets.

⚙️ C++ & Custom Server

The backend is built entirely in C++ without using heavy frameworks like Node.js or Python Flask. It utilizes the WinSock2 API to manually handle HTTP requests.

How it works:

  • The server listens on Port 8080.
  • It intercepts raw TCP packets.
  • It parses the HTTP GET request string.
  • If the request is /search?q=..., it queries the Trie and constructs a JSON response.
  • If the request is for an HTML file, it streams the file content back to the browser.

📄 Dynamic File I/O

To avoid hardcoding, the system uses File I/O streams (`ifstream`) to load the dataset dynamically. Upon startup, the server reads dataset.txt line-by-line, populating the Trie in memory. This makes the system extensible—simply adding a new line to the text file instantly updates the search engine without recompilation.
